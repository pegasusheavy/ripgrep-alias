RIPGREP-ALIAS

Make ripgrep's -E flag work like GNU grep's.
Drop-in. Zero overhead. Done.

THE FLAG CONFLICT

The same flag means two completely different things.

GNU GREP

-E = --extended-regexp

Enables Extended Regular Expressions. You get |, +, ? without backslash escaping. Equivalent to egrep.

RIPGREP

-E = --encoding

Specifies the text encoding of input files — utf-8, latin1, etc. Your regex pattern gets interpreted as an encoding name.

The fix is simple

ripgrep already uses ERE-equivalent syntax by default, so -E is redundant. The wrapper strips it from arguments and forwards everything else to rg untouched. Need the real encoding flag? Use --encoding.

FAQ

A lightweight shell function that wraps rg so that -E and --extended-regexp are silently consumed instead of being misinterpreted as --encoding. Everything else passes straight through to the real binary.
No. The wrapper loops through the argument list in a shell function before calling rg. The overhead is a few microseconds — imperceptible compared to any real search.
Yes. Use the long form --encoding instead of -E. For example: rg --encoding utf-8 'pattern'.
The wrapper strips -E and --extended-regexp as standalone arguments. For combined short flags, separate them: use -i -E instead of -iE.
rg.sh works with any POSIX-compatible shell — bash, zsh, dash, ash, etc. rg.ps1 works with PowerShell 5.1+ and PowerShell Core 7+.
Remove the source line from your shell profile (~/.bashrc, ~/.zshrc, or $PROFILE), then delete ~/.local/share/ripgrep-alias/.
